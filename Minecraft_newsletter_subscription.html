The data set is composed of two files: players.csv includes one observation for each unique player along with demographic information and whether the player subscribed to the newsletter. sessions.csv includes one observation for each play session, including behavioral information such as session length and timing.

Section 1

players.csv

Each row represents a unique player.

  • experience: categorical ordinal player's self-reported, or system-classified, skill level (e.g., Pro, Veteran, Regular, Amateur, Beginner).

  • subscribe: logical / binary whether the player subscribed to the newsletter (True or False). This is the response (target) variable for your project.

  • hashedEmail: string identifier the hashed version of the players email used to link with sessions.csv. No personal information, but repeated values indicate multiple sessions for that player.

  • played_hours: numeric Total number of hours the player has spent on the server. May include outliers or zeros (for inactive players).

  • name: categorical players in-game display name. Some names may repeat or have inconsistent capitalization.

  • gender: categorical players reported gender. May include missing or inconsistent entries.

Section 4

To address the question Can a players experience level predict whether they subscribe to the newsletter?, I plan to use a logistic regression model. This method is suitable for predicting a binary response variable, such as subscribe, using one or more explanatory variables. Logistic regression is interpretable, widely used in introductory data science, and appropriate for the scope of this project.

Logistic regression models the probability of an event, in this case, a player subscribing to the newsletter, conditioned on one or more predictors. The technique can accommodate categorical predictors like experience, which will be translated into binary indicator variables for each level of experience. This way, comparisons between groups of players can be assessed while maintaining the model simple and interpretable.

Assumptions needed:

  • The observations are independent, each players decision concerning subscription is independent of others.

  • The log-odds of the response variable, subscribe, is linearly related to the predictors.

  • No strong multicollinearity between predictors (which is not an issue here since only one predictor is used).

  • Sufficient sample size for each category of experience.

Potential limitations:

  • Logistic regression assumes a simple linear relationship on the log-odds scale and may not capture complex nonlinear behavior.

  • Uneven category sizes in experience might make the estimates less certain for smaller groups.

  • Because the exploratory analysis presented only weak differences between levels of experience, the predictive performance of the model is also limited.

Data processing and evaluation plan:

  • The data should be split into 80% training and 20% testing sets before modeling in order to evaluate performance on unseen data.

  • I would make sure that the split is stratified and maintains similar proportions of subscribers across both sets.

  • Accuracy, ROC-AUC, and confusion matrix metrics would then be used to evaluate the model's performance.

  • If time permitted, I would perform cross-validation on the training set to check the model's stability.

  • Future work may involve adding variables like played_hours or aggregated session statistics, further improving performance.

Conclusion The exploratory analysis above shows that the experience level is only weakly related to the newsletter subscription. Now, the effect of this variable on the likelihood of subscription will be formally tested with a logistic regression model. Although the differences between groups were small, this approach provides a reproducible and interpretable framework to assess the impact of player experience on engagement behaviour.
